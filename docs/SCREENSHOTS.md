- `docs/images/`

## Capture guidance

Keep screenshots clean and consistent:

- use a stable base URL and configured adapter
- avoid half-loaded or transient states
- prefer representative screens over noisy debug states
- crop only if needed for clarity
- keep window size consistent between images
- avoid local personal data, unrelated tabs, or machine-specific clutter

## Refresh guidance

When replacing screenshots later:

- keep the same lowercase filenames when possible
- update the image only if the UI changed meaningfully
- prefer screenshots that reflect the current public ActorHarbor identity
- avoid noisy debug states unless the screenshot is intentionally documenting them
